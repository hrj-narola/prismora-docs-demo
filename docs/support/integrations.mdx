# Integrations (GitHub, Asana, and others)

Prismora can connect to external tools to pull inputs (like code and tasks) and to share outputs (like results and follow-ups).

:::note Typical integration goal
Integrations reduce manual copy/paste by letting Prismora fetch the right inputs and (optionally) create follow-up work automatically.
:::

## Common integration concepts

- Each integration typically requires authorization (sign-in / API token / connection approval).
- Prismora needs to know which resources belong to which project (mapping).
- Some actions run automatically (sync) while others are manual (triggered by you).

## Integration cheat sheet

| Integration | Helps with | Common mapping |
| --- | --- | --- |
| GitHub | Code context for analysis | Repos/branches -> Prismora project |
| Asana | Task follow-ups and work tracking | Asana workspace/project -> Prismora project |
| Others | Pull additional context | Keep mapping consistent with your team scope |

## GitHub

### Use cases

- Bring in code context for analysis and test generation
- Track commits/branches related to changes

### Typical setup

1. Go to Integrations in your Prismora workspace/project settings
2. Choose **GitHub**
3. Authorize the connection
4. Select repositories/branches to connect (if prompted)
5. Save/apply changes

### Troubleshooting

- If Prismora can’t see your repositories, re-check authorization and repository permissions.

## Asana

### Use cases

- Sync tasks and project work
- Turn failing tests into follow-up work items

### Typical setup

1. Go to Integrations
2. Choose **Asana**
3. Authorize the connection
4. Map Asana projects/workspaces to your Prismora project (if prompted)
5. Save/apply changes

### Troubleshooting

- If tasks do not appear in follow-ups, confirm you have the correct Asana workspace/project permissions.

## Other integrations

If your Prismora deployment supports additional integrations, follow the same pattern:

- connect and authorize
- map to the current project
- verify data sync (or run a test sync)
