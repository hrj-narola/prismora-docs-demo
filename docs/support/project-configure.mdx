# Project Configure

This page explains how to configure a Prismora project after you’ve completed basic project setup. Use this when you need to adjust project settings, inputs, and behavior before (or between) test runs.

:::note At a glance
Configuration affects what Prismora uses for analysis, test generation, and test runs.
If something looks “off”, check configuration before re-generating everything.
:::

## What you can configure

- Project metadata (name, description, ownership)
- Which resources are considered inputs for analysis and test generation
- Team access (where applicable)
- Testing preferences (for example: defaults used during generation and runs)

## Configuration checklist

- Confirm the correct inputs are selected (code/requirements/designs/docs)
- Confirm scope matches the feature/release you’re testing
- Confirm any testing preferences (defaults, filters, environments)

## Typical configuration flow

### Steps

1. Open your project
2. Choose **Project Configure** (or the project settings area)
3. Review each configuration section
4. Update values to match your desired workflow
5. Save/apply changes

### Expected result

- Subsequent analysis, test generation, and test runs use the updated configuration.

## Troubleshooting

- If test generation seems inconsistent, confirm the resource inputs selected in configuration match what you expect.
- If project actions are missing or disabled, verify your permissions for that project.
