# Create Project

This page helps you create a new Prismora project and ensure the basics (name, scope, and access) are correct before you upload resources.

## Before you start

- Decide the **project name** and what it should represent (feature, release, or workstream).
- Collect the sources you plan to add next (code changes, requirements, design files, etc.).

## Steps

1. Open your Prismora dashboard.
2. Select **Create Project**.
3. Fill out the required fields:
   - `Project Name` (required): short and descriptive.
   - `Project Description` (recommended): what problem/work this project covers.
   - `Owner / Workspace` (if shown): where this project will live.
4. (If your UI provides it) set access/visibility:
   - `Private` or `Organization` scope (pick what matches your team needs).
5. Click **Create** (or **Save**).

## Expected result

- You land in the newly created project workspace.
- The next action (usually **Upload Resource** or “Add inputs”) is available.

## Common issues

- **Wrong scope**: if later features use the wrong resources, you may need to re-check project access/scope and inputs.
- **Missing metadata**: if you can’t proceed to the next step, verify required fields were completed.
