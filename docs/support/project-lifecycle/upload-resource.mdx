# Upload Resource

Upload the project inputs Prismora needs to understand scope and generate testing artifacts.

## What “resources” mean in Prismora

Typically, a resource is one of the following input types:

- **Code changes**: diffs/commits/patch files or repository selections (depending on your deployment).
- **Requirements**: functional/non-functional requirements, user stories, or acceptance criteria.
- **Design files**: diagrams, flow specs, UI/UX artifacts.
- **Supporting documents**: any additional context your team relies on.

## Steps

1. Open your project.
2. Select **Upload Resource**.
3. Choose the **Resource Type** (code / requirements / design / supporting docs).
4. Provide the content in the format your UI supports:
   - Upload a file
   - Paste text
   - Link/select from an integration (if enabled)
5. Review the uploaded item list (if your UI shows it).
6. Click **Confirm** / **Add** / **Upload**.

## Recommended inputs for best results

- Make sure resource types match what you actually have (don’t upload “requirements” content into a “design” slot).
- Keep your scope consistent with your project goal (for example, only include changes for the target release).

## Common issues

- **Upload fails**: try a smaller file, check supported formats, and ensure encoding is UTF-8 compatible.
- **Analysis later looks wrong**: verify you selected the correct resource type and included the correct artifacts.
