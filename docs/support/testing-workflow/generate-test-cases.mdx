# Generate Test Cases

The **Generate Test Cases** step creates a set of test scenarios based on the inputs configured for your project.

## Choose the input source (important)

Prismora can generate tests from different inputs. Match the source to your goal:

- Code changes (PR/commit diff)
- Requirements
- Design files (flows/states/UI designs)
- Free-form description (“Describe To”)

See: [Generate Test Cases: Input Sources](/docs/support/generate-test-cases-sources)

## What to do

1. Open the testing area.
2. Choose **Generate Test Cases**.
3. Confirm the input source and generation options.
4. Wait for generation to finish.
5. Review the generated set (if your UI provides a review step).

## What to check before you run

- The generated tests reflect the right scope (feature/release boundaries).
- No critical cases are missing (especially from your “must-pass” requirements).
