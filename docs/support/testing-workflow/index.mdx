# Testing Workflow

This section explains the end-to-end testing flow in Prismora: set **Number of test cases**, **Generate Test Cases**, run them with **Test Run**, and review **Test Results**.

:::tip Start here
Use this hub page to navigate the full workflow. Each step has its own page with “what to do” and “what to check”.
:::

## Steps (recommended order)

1. [Number of Test Cases](/docs/support/testing-workflow/number-of-test-cases)
2. [Generate Test Cases](/docs/support/testing-workflow/generate-test-cases)
3. [Test Run](/docs/support/testing-workflow/test-run)
4. [Test Results & Action Buttons](/docs/support/testing-workflow/test-results)

## Quick checklist before you start

- Project setup is finished (you should see testing controls enabled).
- Your inputs are uploaded and analyzed (or you intentionally want to test using the current configuration).
