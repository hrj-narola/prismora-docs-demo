# Number of Test Cases

The **Number of test cases** setting controls how many test scenarios Prismora will generate for the current project scope.

## At a glance

- Higher counts can improve coverage.
- Higher counts also increase runtime and review effort.

:::note Recommendation
Start small (for example: a “smoke set”), then increase once the generated output looks correct.
:::

## What to do

1. Open the testing area for your project.
2. Find **Number of test cases**.
3. Set the target number.
4. Confirm the selection.

## What to verify

- The number you set matches what the UI shows for the current run/generation session.
- You don’t change the inputs mid-way without re-running Generate + Analyze steps.
