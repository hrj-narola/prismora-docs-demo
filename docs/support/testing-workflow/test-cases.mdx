# Test Cases List

Immediately following successful project onboarding, Prismora generates a complete battery of automated tests. You can access these from the **Test Cases** tab located on the left-side main navigation menu.

## Navigating the List

The Test Cases page provides a rich, highly filterable view of all available scenarios generated by the AI Copilot.

### Search and Filters

- **Search Bar**: Quickly locate a specific test case by searching its name, description, or relevant tags.
- **Advanced Filtering**: Use the dynamic dropdowns below the search bar to narrow down the list. You can filter by:
  - Modules (e.g., Cart/Checkout, Store Locator)
  - Types (e.g., Functional, Security, UI)
  - Status (Active vs Inactive)
  - Priority (High, Medium, Low)
  - Custom Groups

### Viewing Test Case Cards

Each test case appears as an individual row providing critical context at a glance:
- **Title and Description**: A clear summary of what the test validates (e.g., "Store Locator Fallback Behavior – Empty or Unavailable Geolocation").
- **Tags**: Colored badges indicating parameters like Status (`Active`), Priority (`High`), Type (`EDGE`, `FUNCTIONAL`, `SECURITY`), and Module (`General`).
- **Timestamps**: The time elapsed since the test case was generated or last executed.

From this list, simply click on any test case row to open its detailed view and review its distinct sequence of test steps.
