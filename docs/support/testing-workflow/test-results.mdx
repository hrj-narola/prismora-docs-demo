# Test Results & Action Buttons

After **Test Run** completes, Prismora shows the outcome of each test and supporting logs/errors (when available).

## At a glance

- Overall status (pass/fail/blocked counts)
- Per-test status and metadata
- Logs and error details (when available)

## “Particular Buttons” you may see

Button names can vary by UI version, but typical actions include:

- `Filter` / `Search` to find failing tests
- `Rerun` to re-execute selected tests
- `View Details` to open a failing test’s information and logs
- `Export` to download results for sharing or reporting
- `Create Follow-up` (or similar) to convert failures into actionable work

:::tip Suggested workflow
Filter to failures first, then open details for the top-priority failing tests.
:::
