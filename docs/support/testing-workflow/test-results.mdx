# Test Results Dashboard

Upon completion of a test run, all historical and analytic data is sent directly to the **Test Results** tab. This dashboard breaks down exactly what happened, simplifying continuous integration assessments.

## AI Insights

At the very top of the results page, the platform will prominently display an **AI Insight** banner if significant patterns are recognized. For example, it will immediately alert you if:
> *1 new regression detected. All are related to functional module.*

## Top-Level Metrics

A summary ribbon provides an instant snapshot of your testing health:
- **Total Cases**: The number of tests executed across associated modules.
- **Passed**: The subset of tests that successfully completed.
- **Failed**: Tests blocked from completion due to errors or unexpected behavior.
- **Skipped / Needs Re-check / Slowest Test**: Insights for optimizing pipeline speed and consistency.

## Detailed Run Analytics

The lower half of the dashboard allows you to interrogate specific test runs.

### Run History

- Select any specific **Test Run** block from the left panel (bearing an ID like `20040849`) to pull up its exact analytics.
- It displays the execution timeline, date, pass/fail percentage, and whether it is actively running or completed.

### Changes Since Last Run

The system contrasts the fresh data against historical runs to populate this panel:
- **New Failures**: Issues that arose in the current execution that previously passed.
- **Fixed**: Issues from a previous run that now execute perfectly.
- **Repeated Issues**: Persistent bugs failing consecutive runs.
- **Needs Re-check**: Sporadic or non-deterministic test results yielding warnings.

You can utilize the master filter row at the bottom of the dashboard (`All Status`, `All Modules`, `All Suites`) to dissect specific components and evaluate the test cases associated with New Failures or Fixed statuses.
